功能说明

正向匿名化

  • 将原始脚本中的表名和字段名替换为无意义的标识符（如 table_001, field_001）
  • 自动删除脚本中的注释和空行（可选）
  • 生成编码字典，用于后续的反向匿名化

反向匿名化

  • 根据编码字典将匿名化后的脚本还原为原始脚本
  • 需要使用与正向匿名化相同的编码字典

示例流程

-- 1. 原始脚本
SELECT user_id, user_name, create_date 
FROM users 
WHERE create_date >= '2024-01-01'

-- 2. 执行正向匿名化后
SELECT field_001, field_002, create_date 
FROM table_001 
WHERE create_date >= '2024-01-01'

-- 3. 编码字典
{
  "tables": {"table_001": "users"},
  "fields": {"field_001": "user_id", "field_002": "user_name"}
}

-- 4. 执行反向匿名化后（还原）
SELECT user_id, user_name, create_date 
FROM users 
WHERE create_date >= '2024-01-01'
