欢迎使用 SQL 脚本转换工具

一个支持多种数据库方言的 SQL 脚本转换工具。将查询中间表的可视化脚本，转换为直接查询原始表的新指标脚本。

功能模块

📊

中间表

管理中间表脚本和可视化脚本，包括脚本的导入、编辑和与中间表的关联管理。

🔒

匿名化

对脚本中的敏感信息进行匿名化处理，保护数据安全，便于分享和调试。

🔄

转换

核心转换功能，创建任务时指定三个脚本的方言，AI 智能分析并生成直接查询原始表的新脚本。

✅

验证

验证转换后的脚本正确性，对比原始脚本和转换后脚本的逻辑等价性，确保转换结果符合预期。

⚙️

配置

系统配置管理，包括 LLM API 密钥、模型参数、数据库方言管理等设置。

核心概念

三种脚本类型

  • 中间表脚本：定义如何从原始表构建中间表的 SQL 脚本
  • 可视化脚本：查询中间表的 SQL 脚本（用于报表、可视化等场景）
  • 转换后脚本：直接查询原始表的新脚本（跳过中间表，性能更优）

方言支持

  • 支持多种数据库方言：MySQL、达梦、PostgreSQL、ORACLE、ODPS 等
  • 创建任务时，可灵活指定三个脚本各自的方言
  • 方言配置存储在数据库中，便于扩展和管理

转换本质

📋
原始查询方式
可视化脚本 → 查询中间表 → 中间表由中间表脚本从原始表构建
↓
🔍
AI 智能分析
分析中间表脚本，理解原始表结构和查询逻辑
↓
✨
转换结果
生成直接查询原始表的新脚本，跳过中间表

快速开始

1

前往 配置 页面设置 LLM API 密钥（用于 AI 智能转换）

2

在 中间表 页面管理中间表脚本和可视化脚本

3

在 转换 页面创建任务，指定三个脚本的方言，执行转换

4

通过 验证 功能对比转换前后脚本的逻辑等价性
