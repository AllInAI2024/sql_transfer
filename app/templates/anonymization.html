功能说明

匿名化规则

  • 表名：自动识别并匿名化为 table_001, table_002 等格式
  • 字段名：自动识别并匿名化为 field_001, field_002 等格式
  • SQL 关键词：所有 SQL 关键词保持不变，支持多种数据库方言
  • 特殊字符串：日期格式字符串（如 'yyyy-MM-dd', 'HH24:MI:SS'）保持不变
  • 排除列表：可配置不需要匿名化的字符串列表，包含这些字符串的名称将保持不变

示例

-- 原始 SQL
SELECT user_id, user_name, create_date 
FROM users 
WHERE create_date >= '2024-01-01'
  AND status = 1

-- 匿名化后
SELECT field_001, field_002, create_date 
FROM table_001 
WHERE create_date >= '2024-01-01'
  AND status = 1

注意：示例中 create_date 没有被匿名化，因为它包含了排除列表中的 "date" 字符串
