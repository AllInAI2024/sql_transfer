匿名化处理

对脚本中的敏感信息进行匿名化处理，保护数据安全。

▶ 匿名化测试

功能说明

匿名化规则

  • 表名：自动识别并匿名化为 table_001, table_002 等格式
  • 字段名：自动识别并匿名化为 field_001, field_002 等格式
  • SQL 关键词：所有 SQL 关键词保持不变，支持多种数据库方言
  • 特殊字符串：日期格式字符串（如 'yyyy-MM-dd', 'HH24:MI:SS'）保持不变
  • 排除列表：可在配置页面设置不需要匿名化的字符串列表

双向匿名化

系统支持正向匿名化和反向匿名化，两次操作均采用同一个编码字典：

  • 正向匿名化：将原始脚本中的表名、字段名替换为无意义的标识符
  • 反向匿名化：根据编码字典将匿名化后的脚本还原为原始脚本

额外功能

  • 删除空行：匿名化过程中自动删除脚本中的空行
  • 删除注释：匿名化过程中自动删除脚本中的注释（包括行内注释和块注释）

示例

-- 原始 SQL
SELECT user_id, user_name, create_date 
FROM users 
WHERE create_date >= '2024-01-01'
  AND status = 1

-- 匿名化后
SELECT field_001, field_002, create_date 
FROM table_001 
WHERE create_date >= '2024-01-01'
  AND field_003 = 1

注意：示例中 create_date 没有被匿名化，因为它包含了排除列表中的 "date" 字符串

配置入口

匿名化相关配置（如排除列表）请在配置页面中进行设置。
