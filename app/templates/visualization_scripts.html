中间表管理

管理中间表脚本和可视化脚本，建立它们之间的关联关系，为后续的脚本转换做准备。

概念说明

📋 中间表脚本

  • 定义如何从原始表构建中间表的 SQL 脚本
  • 独立存在，与任务无关
  • 中间表名全局唯一
  • 可以被多个可视化脚本复用
  • 支持多种数据库方言（MySQL、达梦、PostgreSQL、ORACLE、ODPS 等）

📊 可视化脚本

  • 查询中间表的 SQL 脚本
  • 用于报表、可视化等业务场景
  • 属于某个任务（任务创建时指定方言）
  • 通过 intermediate_table_names 字段关联中间表
  • 一个可视化脚本可关联多个中间表

中间表脚本管理

此页面用于管理中间表脚本，您可以：

  • 创建新的中间表脚本（定义如何从原始表构建中间表）
  • 导入已有的中间表脚本
  • 查看和编辑脚本内容
  • 管理脚本的方言信息

工作流程

  1. 创建中间表脚本：定义如何从原始表构建中间表
  2. 创建可视化脚本：定义查询中间表的业务逻辑
  3. 建立关联：将可视化脚本与所需的中间表脚本关联
  4. 创建转换任务：在转换页面创建任务，指定三个脚本的方言
  5. 执行转换：AI 智能分析并生成直接查询原始表的新脚本
